## 동일한 타입의 변수를 여러 개 사용하면
- 변수의 수 증가
- 코드 길이 증가
- 반복문 적용 불가
- 변수의 수가 동적일 경우, 사용 불가

=> 이러한 문제 해결을 위해 배열을 사용한다


# 배열
> 동일한 타입의 데이터 여러 개를 하나의 **연속된 메모리 공간**에서 관리하는 것

## 특징
- 요소 접근 속도 빠름
- 크기 변경 불가
- 메모리는 heap에 변수 이름은 stack에 저장
  + heap에는 배열 변수의 값
  + stack에는 heap에 저장된 배열의 첫번째 주소 값 저장
- new <데이터 타입> [크기 지정]
